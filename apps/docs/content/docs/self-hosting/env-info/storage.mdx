## Storage Service Environment Variables

### General Settings

#### `DICOM_STORAGE_FILEPATH`

- Type: string (required)
- Description: The path for storing DICOM files, can use parameters `workspaceId`, `0020000D`, `0020000E`, `00080018` and append `hash` parameter (e.g. `{0020000D,hash}`)
- Default: `/dicom/{workspaceId}/{0020000D,hash}/{0020000E,hash}/{00080018,hash}.dcm`
- Example: `/dicom/{workspaceId}/{0020000D,hash}/{0020000E,hash}/{00080018,hash}.dcm`

#### `STORAGE_PROVIDER`

- Type: string (required)
- Description: The storage provider, can be `local` or `s3`
- Default: `local`
- Example: `s3`

### Local Storage Settings

#### `STORAGE_LOCAL_DIR`

- Type: string (required)
- Description: The directory for storing local files
- Default: `./dicom-files`
- Example: `./dicom-files`

### S3 Settings

#### `S3_ENDPOINT`

- Type: string (required)
- Description: The endpoint for the S3 service
- Default: None
- Example: `http://localhost:9000`

#### `S3_BUCKET`

- Type: string (required)
- Description: The name of the S3 bucket
- Default: None
- Example: `brigid`

#### `S3_ACCESS_KEY`

- Type: string (required)
- Description: The access key for the S3 service
- Default: None
- Example: `access_key`

#### `S3_SECRET_KEY`

- Type: string (required)
- Description: The secret key for the S3 service
- Default: None
- Example: `secret_key`
